# BiliFetch 脚本猫版上传说明

脚本猫上传时只需要一个脚本：

- `BiliFetch-ScriptCat.user.js`

项目重心是脚本猫。网站、软件和本地助手都只是服务脚本猫的配套内容。

上传脚本猫时，只上传 `BiliFetch-ScriptCat.user.js`。

脚本猫发布页文案、权限说明和更新日志看：

- `README-脚本猫发布资料.md`

## 2.3.0 核心变化

- 面板顶部标题栏现在可以拖动。
- 拖动后会自动记住位置，下次打开 B 站页面会停在上次位置。

## 2.2.0 核心变化

- 打开 B 站视频页后，面板会自动识别页面内所有视频/分P，并显示成可勾选列表。
- 去掉旧范围下拉框，用户自行勾选要下载的视频。
- 新增“清晰度”选项，默认“最高可用”。
- 选择 1080P、720P 等清晰度时，脚本会自动挑选不超过该清晰度的最好视频流。
- 下载内容改为多选形式。
- 默认勾选“合并视频+音频MP4”。
- 用户可以同时勾选字幕、弹幕、封面、信息 JSON、命令导出等内容。
- 多选后脚本会按勾选的视频和内容逐个处理。
- 本地助手新增任务进度页：`http://127.0.0.1:18991/`。
- 本地助手会检测 ffmpeg 是否能正常运行，并给出修复提示。
- 脚本把任务交给本地助手后，会显示进度页面地址。
- 新增本地助手：启动后，脚本可以把任务发送到本机自动下载并合并。
- 本地助手没启动时，脚本会尝试浏览器内自动合并。
- 合并流程不再自动生成或下载 `.bat`。
- 主界面只保留“开始”和“更多”，预览、诊断、设置放在“更多”里。
- 浏览器内合并依赖 ffmpeg.wasm，首次使用会比较慢；更稳更快的方式是先启动本地助手。
- 增加更多失败原因识别和中文解决建议。
- 脚本会自动判断最合适的路径：
  - 有完整 MP4：直接下载完整 MP4。
  - 音视频分离：优先交给本地助手合并。
  - 本地助手未启动：尝试浏览器内自动合并。
- 合并失败时不会自动下载分离的视频流和音频流。
- 已减少权限：不再申请 `unsafeWindow`。

## 上传步骤

1. 打开脚本猫管理页面。
2. 新建脚本或导入脚本。
3. 粘贴 `BiliFetch-ScriptCat.user.js` 的全部内容。
4. 保存并启用。
5. 打开 B 站普通视频页，例如 `https://www.bilibili.com/video/BV...`。
6. 右下角出现 `BiliFetch 下载助手` 就说明安装成功。

## 用户怎么用

最简单方式：

1. 保持默认勾选的“合并视频+音频MP4”。
2. 清晰度保持默认“最高可用”。
3. 在“选择视频”里确认要下载的视频，默认会勾选当前播放的视频。
4. 点击“开始”。
5. 等脚本自动处理。

如果只想下载 1080P、720P 等清晰度，就在“清晰度”里选择后再点“开始”。

如果还想下载字幕、弹幕、封面或信息 JSON，就继续勾选对应内容，再点“开始”。

更自动的方式：

1. 先解压 `BiliFetch-本地助手.zip`。
2. 双击 `启动BiliFetch本地助手.bat`。
3. 保持本地助手窗口打开。
4. 回到 B 站页面点击“开始”。
5. 文件会保存到 `Downloads\BiliFetch`。
6. 想看进度，打开 `http://127.0.0.1:18991/`。

如果视频是音视频分离：

- 本地助手已启动：自动下载并合并。
- 本地助手未启动：尝试浏览器内自动合并。
- 不会自动下载 `.bat`。

## 重要限制

- 浏览器内自动合并是实验功能，依赖 ffmpeg.wasm，首次使用会比较慢。
- 大视频浏览器内合并可能比较慢，更建议先启动本地助手。
- 大视频可能因为浏览器内存、网络或权限问题导致浏览器内合并失败。
- 浏览器脚本不能自己启动本地程序，所以本地助手需要用户先双击打开。
- 浏览器脚本不能删除用户下载目录里已经存在的文件，所以脚本会尽量避免下载无关文件。
- 脚本不会绕过登录、会员、版权、地区限制。
- 请只下载自己有权保存和使用的内容。
